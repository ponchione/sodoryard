# Task 02: RawChunk and Chunk Structs

**Epic:** 01 — Types and Interfaces
**Status:** ⬚ Not started
**Dependencies:** Task 01

---

## Description

Define the two core data structures of the RAG pipeline. `RawChunk` is the output of a parser — it contains only what the parser can determine from source code. `Chunk` is the fully enriched record stored in LanceDB — it adds identity, metadata, descriptions, embeddings, and relationship data built across all three indexing passes.

## Acceptance Criteria

- [ ] `RawChunk` struct defined in `internal/rag/types.go` with the following fields:
  - `Name      string`    — symbol name (e.g., function name, type name, heading text)
  - `Signature string`    — everything before the body (e.g., `func (s *AuthService) ValidateToken(token string) (*Claims, error)`)
  - `Body      string`    — full body text of the declaration
  - `ChunkType ChunkType` — one of the ChunkType enum values from Task 01
  - `LineStart int`       — 1-based starting line number in the source file
  - `LineEnd   int`       — 1-based ending line number in the source file
- [ ] `Chunk` struct defined in `internal/rag/types.go` with the following fields:
  - **Identity:**
    - `ID          string` — deterministic SHA-256 hash (see Task 04)
    - `ProjectName string` — project identifier for multi-project support
  - **Location:**
    - `FilePath  string`    — path relative to project root
    - `Language  string`    — programming language (e.g., "go", "typescript", "python", "markdown")
    - `ChunkType ChunkType` — syntactic role of this chunk
  - **Content:**
    - `Name        string` — symbol name
    - `Signature   string` — declaration signature (everything before the body)
    - `Body        string` — full body text, truncated to `MaxBodyLength` before storage
    - `Description string` — 1-2 sentence semantic summary generated by the Describer
  - **Metadata:**
    - `LineStart   int`       — 1-based starting line number
    - `LineEnd     int`       — 1-based ending line number
    - `ContentHash string`    — SHA-256 hash of `Body` for change detection
    - `IndexedAt   time.Time` — timestamp of when this chunk was last indexed
  - **Relationships** (all `[]string`; serialized as JSON strings in LanceDB, native slices in Go):
    - `Calls            []string` — qualified names of functions this chunk calls
    - `CalledBy         []string` — qualified names of functions that call this chunk
    - `TypesUsed        []string` — qualified names of types referenced in this chunk
    - `ImplementsIfaces []string` — qualified names of interfaces this chunk's receiver type satisfies
    - `Imports          []string` — package paths imported by the file containing this chunk
  - **Embedding:**
    - `Embedding []float32` — vector embedding of `Signature + "\n" + Description` (3584 dimensions by default)
- [ ] File compiles cleanly: `go build ./internal/rag/...`
